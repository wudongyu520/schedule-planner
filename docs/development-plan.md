# 时间块日程规划系统 - 开发计划与里程碑

## 总览

预计总开发周期：**7-9 周**（业余时间，每天2-3小时）

---

## 里程碑总览

| 阶段 | 名称 | 周期 | 核心产出 |
|------|------|------|----------|
| Phase 0 | 环境搭建与项目初始化 | 3天 | 项目可运行，基础配置完成 |
| Phase 1 | 时间轴与基础布局 | 4天 | 周视图骨架，时间轴展示 |

| Phase 3 | 任务管理与拖拽 | 7天 | 任务CRUD，任务面板，拖拽安排 |
| Phase 4 | 时间线与时间块切换 | 5天 | 当前时间线，自动切换机制 |

| Phase 7 | 用户系统与云端 | 7天 | 注册登录，数据同步 |
| Phase 8 | 部署与优化 | 5天 | Docker部署，性能优化 |

## 学习路径（新手友好）

### 前置知识（1-2周）

如果你对 Next.js / React 不熟悉，建议先学习：

1. **React 基础**（3天）
   - JSX 语法
   - 组件与 Props
   - State 状态
   - Hooks（useState, useEffect）

2. **TypeScript 基础**（2天）
   - 类型定义
   - 接口
   - 泛型

3. **Next.js 基础**（3天）
   - App Router
   - 页面与布局
   - Server/Client Component
   - API Routes

4. **Tailwind CSS**（1天）
   - 常用类名
   - 响应式设计

### 边做边学

每个阶段开始前，会提供：
- 相关知识点讲解
- 代码示例
- 常见问题解答

## 风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 学习曲线陡峭 | 高 | 进度延迟 | 每个阶段前补充学习，提供详细指导 |
| 拖拽实现复杂 | 中 | 进度延迟 | 先实现基础功能，再逐步优化拖拽体验 |
| 时区问题 | 中 | 数据错误 | 统一使用 UTC，显示时转换本地时间 |
| 并发冲突 | 低 | 数据丢失 | 最后写入胜出 + 提示 |
| 性能问题 | 低 | 体验差 | 前期关注性能，后期优化 |

## 开发建议

1. **小步快跑**：每个小功能完成后就提交，不要攒一大堆
2. **先跑通再优化**：先实现功能，再优化体验
3. **遇到问题先搜**：报错信息复制搜索，90% 的问题都有答案
4. **多 console.log**：理解数据流，调试问题
5. **别怕问问题**：不懂就问，节省时间
